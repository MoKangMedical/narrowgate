# 窄门 (NarrowGate) — 让人看见窄门并跨越窄门

> "你们要进窄门。因为引到灭亡，那门是宽的，路是大的，进去的人也多；引到生命，那门是窄的，路是小的，找着的人也少。" — 马太福音 7:13-14

## 这是什么

窄门是一个**进化平台**，不是给人答案的平台，是**制造困境并引导穿越**的平台。

我们训练人一步一步让自己从人变为神——不是宗教意义的神，是**人类潜能的极致释放**。

## 五层进化架构

```
第五层：封神 (Deification) — 创造价值，定义规则，成为他人窄门的守门人
第四层：觉醒 (Awakening) — 看见系统，跳出棋盘，从玩家变架构师
第三层：穿越 (Crossing) — 面对恐惧，打破惯性，在痛苦中重塑自我
第二层：看见 (Seeing) — 识别宽门与窄门，看见自己的枷锁
第一层：凡人 (Mortality) — 承认自己的有限、懒惰、恐惧、自欺
```

## 核心模块

1. **灵魂审计引擎** — AI深度追问，不接受表面答案，直到你看见真相
2. **窄门识别器** — 算法识别你当前最需要突破的点
3. **穿越训练场** — 每日挑战：做你最不想做的那件事
4. **见证者网络** — 每个人需要看见你穿越的人
5. **神性档案** — 你穿越窄门的证据链，这才是真正的简历

## 商业模式

**卖的不是工具，是蜕变结果。**

- 个人版：灵魂审计 + 窄门识别 + 每日挑战
- 穿越版：专属AI教练 + 见证者配对 + 社群
- 企业版：团队进化系统（管理层觉醒训练）
- 封神版：按结果付费 — 构建你的窄门系统

## 技术栈

- 前端：Next.js + Tailwind CSS
- AI引擎：小米MIMO API（核心推理引擎）
- 后端：Python FastAPI
- 数据库：PostgreSQL + Redis
- 部署：腾讯云

## 启动方式

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 配置MIMO API
cp config/settings.example.yaml config/settings.yaml
# 编辑 settings.yaml 填入API密钥

# 3. 启动服务
python src/api/main.py

# 4. 打开浏览器
# http://localhost:8090
```

## 研究背景

窄门平台的底层逻辑，与医疗领域的"鉴别诊断"高度同构：

- **MediChat-RD** 的诊断引擎 → 窄门的"看见"能力
- **MediSlim** 的行为管理 → 窄门的"穿越"能力
- **MediPharma** 的系统构建 → 窄门的"封神"能力

不同领域，同一个底层架构：**识别问题 → 制造困境 → 引导穿越 → 构建系统。**
